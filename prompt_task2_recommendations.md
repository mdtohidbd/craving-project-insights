# Prompt: Recommended Food Component Implementation

## Role
You are an Expert Frontend Developer and UX/UI Specialist.

## Context & Objective
We want to implement a cross-selling feature to increase the average order value. The objective is to create a dynamic "Recommended Food" component on the Menu Details page that suggests complementary items based on the user's current selection (e.g., suggesting a beverage when a burger is viewed).

## Key Requirements

1. **Recommendation Logic:**
   - Implement logic to determine relevant complementary items (e.g., "People who ordered the same also ordered this").

2. **UI Component (Menu Details Page):**
   - Design and build an attractive, non-intrusive Recommended Food component to be displayed on the Menu Details page.
   - Ensure the design aligns with the premium aesthetic of the application (e.g., modern typography, engaging micro-animations).

3. **Cart Integration:**
   - Add a seamless "Add to Cart" interaction directly within the recommendation component.
   - When a user adds a recommended item, the global cart state must update instantly and seamlessly without requiring a page reload.

## Execution Guidelines
- Prioritize a smooth user experience; the addition of a recommended item should feel fast and satisfying.
- Provide the frontend React component code, including necessary state management and styling.
- Explain how the recommendation data will be fetched or computed from the backend.
